# Сценарий видео: Observability and Tracing in LangSmith

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1: Speaker close-up, fading to an animated network graph showing OpenTelemetry spans.** | Welcome to Week 18. Today, we are mastering Observability and Tracing. A black-box LLM isn't enough for production; we must deploy OpenTelemetry, or OTEL, to standardize our tracing. By deploying trace servers, we generate OTEL-spans for every single model call and tool dispatch, making the agent's invisible reasoning completely visible. |
| **Scene 2: Screen recording of the n8n UI, navigating to the log streaming and webhook settings.** | In no-code environments like n8n, enterprise features allow us to export execution logs directly from our visual workflows. By routing these run statistics to destinations like Webhooks or Sentry, we can anonymize audit events for compliance and securely monitor our automation logic. |
| **Scene 3: Side-by-side split screen of LangSmith and Langfuse dashboards showing deep execution trees.** | To truly understand what happens inside an agentic loop, we integrate API tracers like LangSmith and Langfuse. These platforms ingest our trace data and visualize complex execution trees, allowing developers to debug exact inputs, outputs, and reasoning trajectories at every node. |
| **Scene 4: Dynamic dashboard chart highlighting a sudden spike in API token usage.** | Autonomous agents can quickly burn through your API budget if they get stuck in a loop. Cost tracking is absolutely mandatory. By auditing token consumption fees within our raw logs, we can identify inefficient context windows and enforce strict cost-per-task budgets to maintain healthy unit economics. |
| **Scene 5: A stopwatch animation overlaying an API request-response timeline.** | Speed is just as critical as cost. We configure Latency Watchers to track response speeds and Time-To-First-Token, or TTFT. Tracking latency at a granular, per-step level helps us identify infrastructure bottlenecks and ensures that external tool calls aren't dragging down the entire workflow. |
| **Scene 6: A Slack interface where an automated red alert notification pops up.** | Dashboards are useless if no one looks at them. That's why we implement proactive Budget Alerting. We define monitors with explicit thresholds. When an agent experiences a latency regression or a cost spike, a webhook delivers an instant alert directly to your team in Slack or Telegram for immediate triage. |
| **Scene 7: Screen showing VS Code with Python script, highlighting the `@traceable` decorator.** | For custom Python development, we set up programmatic trace hooks. By wrapping your functions with decorators—like `@traceable` from the LangSmith SDK—you automatically generate OTEL-compliant spans. This effortlessly captures inputs, outputs, and runtime exceptions directly into your observability platform. |
| **Scene 8: Speaker close-up, text overlay: "Strong Models!= Reliable Execution".** | Shifting to Harness Engineering, Lecture 1 teaches us a harsh truth: strong models do not guarantee reliable execution. To close the 'Verification Gap'—where agents confidently lie about task completion—we must build robust harnesses that ground the model in reality through strict verification loops. |
| **Scene 9: Architecture diagram showing the 5 agent subsystems and an isolated Docker container.** | Lecture 2 introduces Test Harness structures. An agent without a harness is just a prompt. We must engineer strictly isolated sandboxes and simulate inputs so the agent can safely run integration tests and execute code without risking your production infrastructure. |
| **Scene 10: Split screen comparing a massive 600-line prompt to cleanly divided markdown files.** | Finally, Lectures 3 and 4 dictate that your code repository must be the single source of truth. To prevent 'Instruction Bloat' and the 'Lost in the Middle' effect, never use one massive prompt file. Instead, divide prompts into thematic files and use progressive disclosure to load context only when the agent actively needs it. |
