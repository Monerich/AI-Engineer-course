# Сценарий видео: Case Study: Lead Gen and Cold Outreach Autopilot

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1:** Speaker close-up, transitioning to an Apify actor interface extracting a list of leads from an Apollo search URL into JSON format. | Welcome to Week 8. Direct Apollo API lead extraction is highly expensive, so enterprise architects bypass this by utilizing third-party scraper actors on platforms like Apify. We feed the scraper an Apollo search URL, and it navigates the filters to output an array of raw B2B profiles containing names, LinkedIn URLs, and company domains. |
| **Scene 2:** An n8n workflow showing an HTTP Request node fetching website data, followed by a Markdown converter and an OpenAI node synthesizing a summary. | Raw scraped lists lack the context needed for high-converting emails. We must build a B2B enrichment pipeline. We ping the prospect's company domain, extract the HTML, convert it to clean Markdown to save tokens, and use an LLM to synthesize a structured JSON dossier containing unique selling points and target demographics. |
| **Scene 3:** Split screen comparing a generic cold email ("Hi [Name], I'd like to sell you...") with an AI-generated multi-line personalized icebreaker. | Now we apply AI Personalization. By combining our enriched company dossier with scraped LinkedIn profile data, the LLM generates a highly specific multi-line icebreaker. The goal is "plausible deniability"—writing copy that looks like a human spent 20 minutes researching the prospect, which routinely boosts reply rates to up to 10 percent. |
| **Scene 4:** A visual mapping from an n8n JSON output directly into a HubSpot or Pipedrive REST API node, showing new deals appearing in a CRM pipeline. | Once leads are scraped and enriched, they must be synced to the client's CRM. We utilize native CRM nodes or direct HTTP REST API calls to map our n8n JSON output to custom fields in HubSpot or Pipedrive. This maintains a single source of truth and allows us to route highly qualified leads directly to senior sales reps. |
| **Scene 5:** A funnel diagram showing leads moving from Google Sheets into the Instantly.ai or Smartlead interface, adding to an active campaign queue. | Sending thousands of cold emails through a standard Gmail node will result in immediate domain bans. Enterprise outbound relies on dedicated dispatch infrastructure like Instantly or Smartlead. n8n connects to these platforms via API, pushing the prospect's email and the customized icebreaker directly into pre-warmed campaigns for zero-touch sending. |
| **Scene 6:** A zoomed-out view of the complete n8n canvas, showing the full Directed Acyclic Graph (DAG) from the scheduled trigger to the final email dispatch. | Let's put it into practice. We are building the complete Outreach Engine: Trigger, Scrape, Enrich, Personalize, and Dispatch. By connecting these modules and using Loop nodes to process batches of leads systematically, you create an Autonomous Growth System—an architecture agencies sell for multi-thousand dollar retainers. |
| **Scene 7:** Close-up of a Python Code node with syntax highlighting, showing a `try/except` block parsing messy CSV data and dropping invalid rows. | Raw scraped CSV files are inherently dirty, containing missing values or corrupted strings. Before feeding this data to an LLM, a Python Code node must act as sanitization middleware. By writing defensive Python scripts, we normalize column headers and drop invalid rows, enforcing a clean state handoff. |
| **Scene 8:** An LLM system prompt window highlighting "Spartan, laconic tone of voice" and a list of negative constraints ("Never use the word 'thrilled'"). | Uncontrolled LLMs write using obvious "AI-isms" that ruin conversions. We enforce strict Brand Tone-of-Voice control using Context Engineering. By injecting few-shot examples of perfect human copy and strict negative constraints, we force the AI to write in a blunt, spartan, and highly professional tone. |
| **Scene 9:** A decision tree / n8n Switch node routing an inbound email reply to one of four paths: POSITIVE, NEGATIVE, OOO, or QUESTION. | When replies flood in, managing the inbox manually is unscalable. We build a deterministic workflow to read inbound IMAP responses. An LLM classifies the semantic intent into strict JSON categories. If the intent is NEGATIVE, a Switch node instantly calls the Smartlead API to unsubscribe the user, protecting the sender reputation. |
| **Scene 10:** A dashboard tracking API usage, showing a drop in costs after switching to Claude 3 Haiku for basic tasks, alongside a PostgreSQL metrics database. | Finally, scaling this outreach to thousands of leads requires ruthless token economics. Using the 60/30/10 routing model, we assign simple tasks to cheap models and reserve expensive models only for complex reasoning. With Python telemetry, we track exact token usage, calculating the fractional cent cost per lead to guarantee our ROI. |
