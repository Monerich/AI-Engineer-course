# Сценарий видео: Vector Databases and RAG in Automations

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1:** Speaker in a modern studio. A 3D scatter plot graphic appears, representing words floating in a multi-dimensional vector space. | Welcome to Week 7. Today we unlock true memory for our agents using Semantic Search. Instead of matching exact keywords, we compress text into mathematical vectors called embeddings. Concepts with similar meanings group closely together in this space, allowing us to find relevant data instantly using cosine similarity. |
| **Scene 2:** Split screen showing the logos for Supabase, Pinecone, and Qdrant. | To store these vectors, we deploy specialized Vector Databases. You can spin up serverless indexes using Pinecone, run Qdrant locally, or configure Supabase with the pgvector extension to store high-dimensional embeddings right alongside your standard relational database tables. |
| **Scene 3:** Screencast of an n8n workflow. A document icon is sliced by a scissor icon (Recursive Character Text Splitter) before entering an OpenAI embeddings node. | In n8n, data ingestion requires strict pipelines. You cannot feed a 50-page PDF directly to an LLM. We use a Recursive Character Text Splitter to slice documents into manageable chunks, embed them into numbers, and upsert them into our database with metadata. |
| **Scene 4:** Close-up of the n8n Vector Store Retriever node settings. The cursor adjusts the relevance score threshold. | When a user asks a question, we use the Vector Store node as a retriever. It vectorizes the query and performs a nearest-neighbor search. Crucially, we set strict relevance thresholds—like requiring a score greater than 0.4—to drop garbage data before it ever reaches the AI. |
| **Scene 5:** A chat interface showing a bot perfectly answering a complex warranty question based on an uploaded corporate PDF. | Let's put this into practice by building a Product Knowledge Base. We equip an AI agent with our retriever tool and a strict system prompt: "Only answer using relevant knowledge provided to you. Do not make anything up." Your agent now acts as a flawless, hallucination-free expert on your company data. |
| **Scene 6:** An animation showing a list of 20 search results dynamically re-sorting themselves based on a secondary scoring pass. | Standard vector searches are incredibly fast, but they are approximate. To achieve ultimate precision, we integrate the Cohere Rerank API. A Reranker acts as a sophisticated secondary judge, cross-evaluating your top results and pushing the absolute best matches to the top of the context window. |
| **Scene 7:** A dark-themed code editor typing out a Python script using `openai.embeddings.create` inside a `try/except` loop. | While n8n is great, processing 50,000 corporate documents requires Python. We programmatically generate text embeddings using the OpenAI API. By writing custom Python harnesses, we handle batching and use exponential backoff to safely navigate strict API rate limits. |
| **Scene 8:** A LangSmith dashboard showing metrics like "Groundedness" and "Context Relevance" with green checkmarks. | How do you mathematically prove your RAG system works? You build RAG Evals. We deploy powerful LLMs as judges to measure Context Retrieval Quality. This evaluates if the retrieved chunks actually answer the prompt, ensuring quality before you push your workflow to production. |
| **Scene 9:** A massive block of dense text is shown. A warning flashes, and the text compresses into three highly focused bullet points. | If you dump too many retrieved chunks into your prompt, your agent suffers from Contextual Degradation, known as the "Lost in the Middle" effect. We overcome this by utilizing context compression—having sub-agents summarize retrieved data so the primary agent stays focused on the goal. |
| **Scene 10:** Two search streams (Dense Vectors and Sparse Keywords) run in parallel, fusing together into a single "Hybrid" result pipeline. | Finally, pure vector search struggles to find exact IDs or error codes. For enterprise applications, we design Hybrid Search. By mathematically fusing dense vector embeddings with sparse BM25 keyword search, your agent receives context that is both semantically rich and lexically exact. |
