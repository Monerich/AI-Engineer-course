# Сценарий видео: Security, Injections and Sandboxes

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1:** Speaker close-up with an animated threat matrix diagram categorizing LLM vulnerabilities. | Welcome to Week 21: Security, Injections, and Sandboxes. Today, we categorize LLM threats into two domains: intrinsic vulnerabilities, like jailbreaks inside the model's brain, and extrinsic risks, like malicious data in your agent's memory. As autonomy scales, so does the attack surface, requiring comprehensive threat mitigation. |
| **Scene 2:** An n8n webhook node passing incoming data through a visual "Sanitization Filter" block. | In Block 2, we secure the perimeter. Indirect prompt injections often sneak in through external webhooks or retrieved RAG documents. We must implement rigorous input sanitization, treating all incoming webhook payloads as hostile until a filter neutralizes malicious patterns before LLM processing,. |
| **Scene 3:** Split screen showing an attacker terminal trying to extract data versus a locked System Prompt Vault. | Block 3 tackles System Prompt Protections. Attackers constantly try prompt stealing attacks to extract your agent's hidden, defining instructions and business logic,. We counter this by structuring prompt architectures with strict context boundaries, ensuring core instructions never leak to the end user. |
| **Scene 4:** A high-contrast OWASP Top 10 checklist graphic appearing on screen with green checkmarks. | Block 4 bridges theory and practice using the OWASP Top 10 for LLM Apps. This is your production baseline. We enforce absolute rules: sanitize all user inputs, never trust an LLM output for irreversible actions, and utilize secure credential systems like the n8n vault rather than committing keys to GitHub. |
| **Scene 5:** Diagram showing an LLM centered with layered protective shields (Llama Guard / NeMo). | For Block 5, we set up Guardrails. We layer semantic and syntactic checks to validate outputs for factual accuracy and safety. By concurrently running relevance classifiers and PII filters, we instantly flag off-topic queries or unnecessary exposure of personally identifiable information,. |
| **Scene 6:** Database schema showing locked tables and a restricted API key vault. | Block 6 focuses on Database Permission Capping. We enforce the principle of least privilege to ensure agents only have the permissions necessary to complete their specific tasks. We isolate roles to prevent agents from executing unauthorized actions like catastrophic data deletions or malicious data extraction. |
| **Scene 7:** A Python terminal executing code securely inside an isolated, glowing container box. | In Block 7, we handle dynamically generated code. The golden rule is to never execute model outputs directly in your main host process. We route all untrusted Python code into ephemeral, isolated sandboxes like E2B, Daytona, or Modal, destroying them instantly to prevent host compromise. |
| **Scene 8:** Flowchart showing a dual-classifier Auto Mode pipeline filtering dangerous requests. | Block 8 explores security-first operating system paradigms. Drawing from advanced Auto Mode architectures, we use fast transcript classifiers to gate overeager tool calls. We also implement prompt-injection probes to screen external tool results before they bypass safeguards and pollute the main agent's context. |
| **Scene 9:** Network VPC diagram showing explicitly blocked outbound connections and proxy filters. | Block 9 locks down communication. A sandbox is useless if it can freely access your internal network. We engineer custom network policies with strict boundaries, ensuring agents cannot exfiltrate data, interact with host credentials, or perform server-side request forgery across your VPC,. |
| **Scene 10:** A bold red 'Approve / Deny' prompt overlaying a transaction workflow timeline. | Finally, Block 10 implements the ultimate fail-safe: Human-in-the-loop triggers. For any irreversible, destructive, or high-risk action, we enforce obligatory manual confirmation,. Using mechanisms like LangGraph interrupts or Claude permission prompts, we keep human oversight firmly in control of the agent. |
