# Сценарий видео: Context Engineering and Modern Prompting Methods

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1:** Speaker in a modern studio. Graphics overlay shows 'Prompting 2023' crossing out and fading into 'Context Engineering 2026' with visual token flows. | Welcome to Week 2. In 2026, prompt engineering as a standalone skill is dead. The modern paradigm is Context Engineering. It is no longer about finding "magic phrases" to trick the AI; it is about answering what exact configuration of context generates the model's desired behavior. You must master token economics and actively manage the model's attention budget to build production-ready systems. |
| **Scene 2:** Screen split. Left: Speaker. Right: Code editor zooming in on a `role: "system"` block containing strict JSON formatting instructions. | Everything starts with System Instructions. This is where you define the agent's foundational persona and lay down strict boundaries. Instead of just asking for a summary, you use the system prompt to enforce strict output structures, like commanding the model to exclusively output valid JSON. This establishes programmatic control over the language model. |
| **Scene 3:** Animation showing raw text colliding with a barrier. Then, XML tags like `<user_data>` and `<instructions>` safely envelope the text. | To protect those boundaries, we use XML Tag Prompting. Modern models are highly optimized to parse and adhere to information inside XML tags. By wrapping variables and inputs in specific tags, you effectively isolate the context. This creates a hard boundary between your trusted instructions and untrusted user data, preventing prompt injection. |
| **Scene 4:** A decision tree graphic on screen. The LLM processes data. Path A: High Confidence. Path B: Uncertainty triggers a fallback text mechanism. | Even with strict boundaries, models can hallucinate. To mitigate this, you must explicitly program how the agent handles uncertainty. In your instructions, give the model a clear fallback rule—for example, "If the answer is not in the text, say 'I do not know'". This simple rule closes the verification gap and stops the agent from fabricating facts. |
| **Scene 5:** Screencast of an n8n workflow. A messy, unstructured email enters an LLM node, and a perfectly structured JSON object emerges inside `<response>` tags. | Let's put this into practice with a JSON Lead Extractor. This requires combining all our methods. We give the agent a system prompt to act as an extractor, provide an XML-tagged example of ideal output, and mandate that it wraps the final JSON payload strictly inside `<response>` tags,. This turns chaotic inbound emails into clean database arrays. |
| **Scene 6:** Warning icons flashing on a system prompt document. A shield icon labeled "Prompt Guard" activates and blocks the threat. | But what happens when users try to hack your agent? Context Security is about designing a Prompt Guard. Attackers will use jailbreaks to try and steal your system prompt. To defend against this, formulate rules using positive actions rather than prohibitions, and prioritize security directives to prevent malicious instruction overrides. |
| **Scene 7:** Python IDE typing out a `try/except` block, safely parsing a JSON string with the `json.loads()` method. | Relying solely on the LLM's output is dangerous. That's why we use Python to bridge the gap. We write Python scripts to read the raw text, extract the tags, and parse the data into dictionaries. By wrapping this According to the sources, and `except` blocks, we gracefully handle `JSONDecodeError` exceptions if the model hallucinates the syntax. |
| **Scene 8:** A 4-quadrant grid lighting up sequentially: Write, Select, Compress, Isolate. | As your data scales, dumping everything into the prompt causes Instruction Bloat. To manage massive payloads, we rely on four Context Engineering primitives: Write, Select, Compress, and Isolate. Instead of overwhelming a single agent, you isolate contexts and compress data, ensuring the model's attention budget remains highly focused. |
| **Scene 9:** Code showing `<example>` tags, transitioning to a model generating text inside `<thought>` tags before outputting the final answer. | To guarantee precision, we use Few-Shot prompting with dynamic example selection. Providing Claude with exact examples is the most effective way to steer its behavior. We pair this with Precognition, or Chain of Thought. By forcing the model to think step-by-step inside `<thought>` tags, it calculates its logic before committing to a final answer. |
| **Scene 10:** A cyclical ReAct loop animation: Observe -> Think -> Act. The loop pauses as a tool is executed by the system, then resumes. | Finally, we arrive at the ReAct loop—Reason and Act. This is the orchestration engine that turns an LLM into an autonomous agent. The agent observes the prompt, thinks about its next step, and requests an action. Your harness intercepts this, executes the real-world tool, and feeds the observation back. Master this loop, and you master Agentic AI. |
