# Сценарий видео: n8n Foundations and Your First Workflow

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1: Presenter on camera, transitioning to a terminal screen showing Docker compose logs.** | Welcome to Week 3. We begin with n8n infrastructure. For production, self-hosting n8n via Docker alongside a PostgreSQL database is the standard. Self-hosting gives you absolute control over your data and bypasses cloud execution limits. |
| **Scene 2: Screen recording of the n8n canvas, highlighting the 'Add first step' panel and trigger types.** | Every n8n workflow starts with a Trigger on the visual canvas. Triggers listen for Webhooks, App events, or run on a Cron schedule. Once triggered, data flows sequentially into Action nodes that execute your business logic. |
| **Scene 3: Close-up of an n8n node configuration panel, typing an expression with `$json`.** | Workflows require dynamic data. Instead of static text, we use n8n's Expression syntax. By writing `$json` followed by JavaScript dot notation, you can dynamically reference and inject data from any previous node directly into your current step. |
| **Scene 4: Visual graphic showing an n8n workflow branching into three paths and merging back.** | Real logic requires branching. We use IF and Switch nodes to route data down distinct conditional paths based on specific rules. Later, Merge nodes allow you to combine these separate execution branches back into a single unified stream. |
| **Scene 5: Split screen: Webhook receiving a payload on the left, Gmail node sending an email on the right.** | Let's practice. We can build an automated welcome pipeline by intercepting a form submission via webhook. The workflow parses the lead's data, logs any missing fields, and triggers a Gmail action node to send a personalized welcome email. |
| **Scene 6: Presenter screen, showing the 'Credentials' tab in n8n with a blurred API key.** | Security is non-negotiable. You must never hardcode API keys or Bearer tokens directly into HTTP headers. Instead, n8n uses a dedicated Credentials vault to securely store and inject keys and OAuth2 tokens into your nodes. |
| **Scene 7: Python code snippet highlighting `def`, type hints, and a `try-except-finally` structure.** | When basic nodes aren't enough, we use Python. You must use type hints to ensure your AI agents pass the correct data formats. Furthermore, wrap your logic According to the sources, blocks to gracefully catch crashes and release system resources. |
| **Scene 8: Diagram showing a failed node routing to an LLM evaluator, then looping back.** | In production, APIs will fail. We combat this by activating 'Retry On Fail' for temporary network shocks. For critical failures, we route data to a global Error Workflow that logs the crash and uses diagnostic loops to self-heal. |
| **Scene 9: Screen showing a 'Wait' node pausing a workflow, alongside a database node saving state.** | Linear pipelines execute quickly, but complex agents require State Transitions. For long-running tasks, you must save the execution context to a database checkpoint and use Wait nodes to suspend the workflow indefinitely until a human approves the action. |
| **Scene 10: Presenter on camera, graphics showing a linear DAG vs a cyclic LangGraph loop.** | Finally, understand the limits of no-code. Platforms like n8n are Directed Acyclic Graphs, perfect for deterministic, linear workflows. When your cognitive architectures demand deep, unpredictable cyclic loops, you must transition to code-heavy frameworks like LangGraph. |
