# Презентация: Голосовые и Мультимодальные Агенты

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Инфраструктура медиа-стриминга
* **Visual Layout Concept:** Тёмная тема с неоново-синей топологией сети. Иконки Lucide: `Radio` и `Server`.
* **Key bullet points:**
 * Переход от традиционных REST API (stateless) к постоянным stateful-соединениям для сохранения контекста.
 * Управление непрерывными бинарными потоками данных вместо статических JSON-ответов.
 * Понимание компромиссов протоколов TCP и UDP для потокового аудио в реальном времени.

📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Телефония с ИИ
* **Visual Layout Concept:** Светлая тема, разделенный экран: традиционное дерево IVR против API-шлюза. Иконка Lucide: `PhoneCall`.
* **Key bullet points:**
 * Подключение традиционной телефонии к ИИ-сетям через протокол SIP и медиа-потоки Twilio.
 * Обработка вебхуков для триггеров входящих звонков и настройка исходящего обзвона.
 * Управление и транскодирование форматов аудио с низкой частотой дискретизации (например, 8kHz `g711_ulaw`).

📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Голосовые сценарии
* **Visual Layout Concept:** Тёмная тема с динамической блок-схемой адаптивных узлов принятия решений. Иконки Lucide: `GitMerge` и `Mic`.
* **Key bullet points:**
 * Замена статических DTMF-меню на динамическую маршрутизацию вызовов на основе намерений пользователя.
 * Использование LLM в качестве агентов первичной сортировки (Triage) для маршрутизации к специализированным суб-агентам.
 * Выполнение поиска данных в CRM и базах знаний в реальном времени на основе устных ответов.

---
📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Комнаты LiveKit
* **Visual Layout Concept:** Высококонтрастная светлая тема со схемой WebRTC (peer-to-peer). Иконки Lucide: `Video` и `Activity`.
* **Key bullet points:**
 * Настройка Selective Forwarding Units (SFU) для эффективной маршрутизации медиа-треков.
 * Использование протокола WebRTC для соединений с ультранизкой задержкой (<300 мс) поверх UDP.
 * Управление состояниями комнат, участниками и автоматической подпиской на нужные аудиотреки.

---
📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Трансляция звука
* **Visual Layout Concept:** Тёмная тема в стиле терминала, отображающая поток строк base64. Иконки Lucide: `Binary` и `Plug`.
* **Key bullet points:**
 * Установка полнодуплексных TCP-туннелей (WebSockets) для двунаправленной медиа-коммуникации.
 * Нарезка аналогового звука на стандартные фрагменты `pcm16` 24kHz.
 * Кодирование сырых аудиопотоков через base64 для передачи на сервер и обратное декодирование.

---
📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Сервисы STT/TTS
* **Visual Layout Concept:** Светлая тема с каскадной трехступенчатой блок-схемой. Иконки Lucide: `MessageSquare` и `Volume2`.
* **Key bullet points:**
 * Построение каскадных конвейеров: STT транскрибирует, LLM формирует ответ, TTS синтезирует звук.
 * Идентификация главного недостатка каскадных систем: накопление задержки (Latency Accumulation, 1.5–3.5 сек).
 * Неизбежная потеря паралингвистических данных (эмоции, интонация, ритм) при использовании текстовых посредников.

---
📊 Слайд 7. Блок 7 (Python-разработка): Разработка скриптов реального времени
* **Visual Layout Concept:** Тёмная тема со сниппетами кода на Python. Иконки Lucide: `Code` и `Bot`.
* **Key bullet points:**
 * Восприятие процесса на Python как "операционной системы" для агента.
 * Внедрение фреймворка DOE (Directive, Orchestration, Execution) для детерминированной надежности пайплайна.
 * Оркестрация `VoicePipelineAgent` с модульным подключением плагинов для VAD, STT и TTS.

📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Реализация голосового общения
* **Visual Layout Concept:** Яркая тёмная тема с прямой связью пользователь-модель (символ бесконечности). Иконки Lucide: `Zap` и `Cpu`.
* **Key bullet points:**
 * Переход к нативным Omni-моделям (OpenAI Realtime API), обрабатывающим аудио напрямую в обход текста.
 * Инициализация строгих границ сессии, системных промптов и схем инструментов до начала общения.
 * Сохранение эмоционального тона и достижение задержки до первого байта аудио (TTFB) менее 300 мс.

---
📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Оптимизация задержек передачи звука
* **Visual Layout Concept:** Светлая тема, отображающая конвейер `asyncio.Queue`. Иконки Lucide: `Gauge` и `Database`.
* **Key bullet points:**
 * Внедрение Jitter-буферов для сглаживания сетевой энтропии и разброса времени прибытия пакетов.
 * Решение проблемы Head-of-Line Blocking в TCP путём агрессивного отбрасывания устаревших аудиофреймов.
 * Управление «гниением контекста» (Context Rot) через фоновую компакцию токенов при длинных сессиях.

---
📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Динамическая обработка перебиваний
* **Visual Layout Concept:** Тёмная тема с предупреждающими красно-оранжевыми акцентами. Иконки Lucide: `ShieldAlert` и `XOctagon`.
* **Key bullet points:**
 * Использование серверного VAD для мгновенного обнаружения прерываний (barge-in).
 * Перехват события `speech_started` для мгновенной очистки очередей воспроизведения на фронтенде.
 * Безопасная отмена задач: хирургическое прерывание функций чтения (`task.cancel()`) и атомарная защита транзакций записи в БД (`asyncio.shield()`) для избежания разрыва верификации.
