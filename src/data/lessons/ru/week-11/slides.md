📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Логика канонического цикла рассуждений ReAct (Think -> Act -> Observe).
**Visual Layout Concept:** Light mode, circular loop diagram (Thought -> Action -> Observation). **Lucide Icons:** `RefreshCw`, `BrainCircuit`.
* Агент работает как `while True` цикл, который объединяет логические рассуждения (Thought) и физические действия (Action).
* Модель анализирует контекст, решает вызвать инструмент, дожидается результата (Observation) и корректирует свой дальнейший план.
* Цикл прерывается только тогда, когда модель возвращает специальный флаг завершения задачи.

📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Парсинг структуры вызовов инструментов (tool_calls) и возврат результатов в историю.
**Visual Layout Concept:** Dark mode (HSL: 200, 15%, 15%), payload transformation graphic. **Lucide Icons:** `Download`, `MessageSquare`.
* Обвязка (harness) перехватывает триггеры LLM (`tool_calls`), физически выполняет функции и возвращает форматированный `tool_result` обратно в массив.
* Результаты более 20K токенов обрезаются или выгружаются в файловую систему (Filesystem offload), чтобы не переполнить контекст.
* Обработка ошибок встроена в этот слой, отправляя диагностические инструкции агенту для автокоррекции.

📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Ограничение шагов итераций для защиты агента от бесконечного зацикливания.
**Visual Layout Concept:** Light mode, warning thresholds and circuit breaker design. **Lucide Icons:** `StopCircle`, `AlertTriangle`.
* Из-за вероятностной природы LLM агенты могут попасть в "мертвые петли" (doom loops), бесконечно повторяя один и тот же ошибочный подход.
* Обязательна установка жесткого лимита итераций (например, 10–15), который принудительно останавливает цикл и экономит API-бюджет.
* Использование мягких ограничений (LoopDetectionMiddleware) позволяет внедрять в контекст предупреждения, заставляя агента изменить стратегию до жесткого отключения.
