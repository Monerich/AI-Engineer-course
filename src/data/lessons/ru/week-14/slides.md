# Презентация: Продвинутые команды и кастомные инструменты в CrewAI

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Инструменты доступа к БД — обертка аналитических запросов к БД в инструменты агентов.
**Visual Layout Concept:** Тёмная тема (Dark mode) с контрастными зелеными акцентами. Иконки Lucide: `Database`, `ShieldCheck`, `Code`.
* **Архитектура NL2SQL:** Итеративный конвейер, где агент переводит естественный язык в SQL, выполняет запрос и синтезирует финальный ответ.
* **Безопасная песочница (Harness):** Обертка инструмента отклоняет деструктивные команды (DROP, DELETE) через regex-фильтры и использует Read-Only доступы.
* **Сжатие контекста:** Принудительное добавление `LIMIT` (например, `LIMIT 50`) предотвращает переполнение окна контекста и ошибку HTTP 400 при запросах к массивным таблицам.
* **Цикл самокоррекции:** Инструмент перехватывает синтаксические SQL-ошибки и возвращает их агенту в виде текста для автоматического исправления.

---

📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): CRM-инструменты — API-интеграции для работы мультиагентных команд с CRM-системами.
**Visual Layout Concept:** Светлая тема (Light mode), схема потока данных (API Flowchart). Иконки Lucide: `Briefcase`, `Network`, `Webhook`.
* **CRM-Автопилот:** Подключение агентов к API систем вроде HubSpot, Pipedrive или Bitrix24 для устранения рутинного ввода данных.
* **Извлечение сущностей:** LLM анализирует транскрипты встреч, извлекает ключевые поля (боли клиента, ЛПР, сроки) и автоматически обновляет стадию сделки.
* **Изоляция доступов:** Использование MCP-серверов или платформ (например, n8n) для хранения API-ключей CRM вне контекста самой языковой модели.

---

📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Продвинутый обмен данными — подготовка промежуточных данных для обмена.
**Visual Layout Concept:** Схема преобразования текста в JSON. Иконки Lucide: `ArrowRightLeft`, `FileJson`.
* **Изоляция и Сжатие:** Агент-рисерчер не передает сырые данные поиска родительскому узлу; он сохраняет их в виртуальную ФС и возвращает сжатое саммари.
* **Чистая передача (Clean Handoff):** Принудительное формирование промежуточных результатов в виде строгих Pydantic/JSON схем для передачи следующему агенту в цепи.
* **Устранение Verification Gap:** Строгая типизация межагентных сообщений гарантирует, что следующий агент не сломается из-за галлюцинаций в структуре данных.

---

📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Защита API от блокировок — управление одновременной нагрузкой от нескольких агентов.
**Visual Layout Concept:** Темная тема, диаграмма сетевой очереди и балансировщика. Иконки Lucide: `Shield`, `Timer`, `Activity`.
* **Проблема Thundering Herd:** Параллельный запуск 50+ суб-агентов может мгновенно вызвать ошибку `HTTP 429 Rate Limit` от внешних API.
* **Асинхронные слушатели и батчинг:** Внедрение очередей задач (например, узлы Split in Batches или Limit) для дозирования исходящих запросов.
* **Durable Execution:** Использование механизмов повторных попыток (retries) с экспоненциальной задержкой, чтобы защитить конвейер от падения при временной недоступности API.

---

📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Экономика CrewAI — анализ затрат и оптимизация стоимости одного запуска сложной команды.
**Visual Layout Concept:** Дашборд с графиками затрат (HSL: 210, 20%, 20%). Иконки Lucide: `DollarSign`, `TrendingDown`.
* **Налог на мультиагентность:** Обмен контекстом между агентами увеличивает расход токенов до 15 раз по сравнению с одиночным вызовом.
* **Маршрутизация моделей (Model Routing):** Использование дорогих моделей (Claude 3.5 Opus) только для Оркестратора, а дешевых (Haiku / GPT-4o-mini) для рутинных воркеров.
* **Prompt Caching:** Агрессивное кеширование системных промптов, описаний инструментов и общих инструкций () экономит до 90% стоимости повторяющихся задач.

---

📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Экспорт отчетов — экспорт результатов выполнения команды в PDF и Markdown.
**Visual Layout Concept:** Анимация сборки документа: от шестеренок к готовому файлу. Иконки Lucide: `FileText`, `Download`.
* **Метод Карпатого:** Разделение хранилища на сырые данные (`raw/`), базу знаний (`wiki/`) и финальные отчеты (`outputs/`) для сохранения чистоты контекста.
* **Генерация артефактов:** Агенты собирают итоговые Markdown-отчеты с инлайн-цитатами, которые скрипты обвязки автоматически конвертируют в HTML или PDF.
* **Дашборды:** Динамическая выгрузка финансовых или аналитических метрик в Excel-файлы или UI-панели для конечного клиента.

---

📊 Слайд 7. Блок 7 (Python-разработка): Сохранение состояний мультиагентных систем с использованием внешних баз данных PostgreSQL/SQLite.
**Visual Layout Concept:** Архитектура сохранения состояния (State Machine -> DB). Иконки Lucide: `Server`, `HardDrive`.
* **ACID для агентов:** Сохранение состояния агента (память, логи сессии) должно подчиняться принципам транзакций БД.
* **Checkpointers:** Интеграция `PostgresSaver` или `SqliteSaver` для фиксации состояния графа выполнения после каждого узла.
* **Durable Execution:** Защита от «амнезии» при падении облачных контейнеров; возможность приостановить (interrupt), перемотать (rewind) или форкнуть процесс.

---

📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Разрешение логических противоречий и конфликтов между выводами разных агентов.
**Visual Layout Concept:** Темная тема, иерархия с Агентом-Судьей на вершине. Иконки Lucide: `Swords`, `Scale`.
* **Мультиагентный консенсус:** Параллельный запуск агентов для создания "комнаты для дебатов", где статистический разброс идей повышает итоговое качество ответов.
* **Арбитраж (The Judge):** Оркестратор с нулевой температурой анализирует противоречивые логи суб-агентов (например, Оптимиста и Пессимиста) и выносит детерминированный вердикт.
* **Верификация кодом:** Лучший способ разрешить спор двух агентов о коде — запустить сквозной (E2E) тест в песочнице. Побеждает рабочий код.

---

📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Цепочки контроля качества: автор (Writer) -> верификатор (Verifier) -> редактор (Editor).
**Visual Layout Concept:** Линейный конвейер с изолированными блоками. Иконки Lucide: `PenTool`, `CheckCircle`.
* **Паттерн Actor-Critic:** Один агент пишет черновик, другой (Верификатор) ищет фактические ошибки, третий (Редактор) устраняет дефекты.
* **Нулевая предвзятость (Zero Bias):** Верификатор запускается с абсолютно чистым контекстом, не содержащим логику Автора, чтобы избежать когнитивного искажения подтверждения.
* **Красные пометки (Red Marks):** Верификатор выдает не просто статус "Провал", а конкретный JSON с указанием: что не так, почему и как это исправить.

---

📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Системы метрик и автоматическая оценка качества работы сложных мультиагентных групп.
**Visual Layout Concept:** Дашборд CI/CD пайплайна (Fail/Pass metrics). Иконки Lucide: `LineChart`, `Target`, `GitMerge`.
* **Четыре типа Evals:** Внедрение одношаговых (Single-turn), траекторных (Trajectory), LLM-as-a-Judge и оценок конечного состояния (End-state).
* **Golden Datasets:** Сбор 30-50 сложных продакшен-сценариев с размеченными эталонными ответами для постоянного регрессионного тестирования.
* **CI/CD Gating:** Автоматическая блокировка слияния пулл-реквеста, если обновление промпта или инструмента снижает Pass Rate мультиагентного роя (интеграция с LangSmith / Braintrust / Inspect).
