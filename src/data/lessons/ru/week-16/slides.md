# Презентация: Долгосрочная память и Человек в контуре в LangGraph

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Чекпоинты графа — подключение Postgres и SQLite для долговременного сохранения чекпоинтов.
**Visual Layout Concept:** Темная тема HSL (`hsl(222, 47%, 11%)`). Иконки Lucide: `Database`, `Save`. Схема персистентности, показывающая сохранение состояния графа в базу данных после каждого узла.
* В LangGraph долговечное исполнение (durable execution) обеспечивается встроенным слоем персистентности (чекпоинтами). 
* Вы можете использовать легковесный SQLite для тестирования или `PostgresSaver` для производственных нагрузок, чтобы сохранять состояние агента после каждого шага.
* Это предотвращает потерю данных при критических сбоях и позволяет ставить длительные задачи на паузу, надежно возобновляя их в будущем.

---

📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Роутинг сессий — идентификация сессий и роутинг пользователей при работе с графом.
**Visual Layout Concept:** Светлая тема HSL (`hsl(210, 40%, 98%)`). Иконки Lucide: `Route`, `Users`. Диаграмма маршрутизации, разделяющая входящие запросы по `user_id` и `thread_id`.
* Для работы в продакшне необходимо жестко разделять контексты множества пользователей.
* Память должна быть концептуально разделена на потоковую (`thread-scoped`) для текущей беседы и пользовательскую (`user-scoped`) для фактов, сохраняющихся между сессиями. 
* Передавая уникальный идентификатор сессии (`thread_id`) в конфигурацию рантайма, вы маршрутизируете исполнение и гарантируете, что граф подхватит ровно то состояние, на котором пользователь остановился.

📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Мониторинг графа — создание панелей отслеживания истории переходов и состояний.
**Visual Layout Concept:** Темная тема HSL (`hsl(222, 47%, 11%)`). Иконки Lucide: `Activity`, `Eye`. Макет дашборда LangSmith, демонстрирующий трейсы выполнения (spans) и задержки.
* Без должной наблюдаемости агенты принимают решения в условиях неопределенности, а повторные попытки (retries) превращаются в слепое блуждание.
* Интегрируйте OpenTelemetry (OTEL) трассировку, чтобы фиксировать каждый вызов модели, инструмента и запуск суб-агента.
* Платформы вроде LangSmith позволяют визуально отслеживать потребление токенов, задержку (latency) и точные переходы состояний от узла к узлу внутри LangGraph.

📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Обогащение контекста — интеграция с внешними базами знаний для обогащения состояния.
**Visual Layout Concept:** Светлая тема HSL (`hsl(0, 0%, 100%)`). Иконки Lucide: `Library`, `BrainCircuit`. Схема работы RAG: пользовательский запрос -> поиск в векторе -> обогащение состояния графа.
* Архитектуры Retrieval-Augmented Generation (RAG) расширяют знания модели за пределы ее базового обучения, подключая структурированные и неструктурированные хранилища данных.
* Агенты могут использовать инструменты для поиска по векторным базам (например, Qdrant, Supabase Vector) и извлекать релевантный контент [10-12].
* Обогащайте объект `State` в LangGraph этими фактами перед передачей в узел LLM, чтобы агент генерировал ответы на основе актуальных и достоверных бизнес-данных.

📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Безопасность сессий — сохранение учетных данных пользователей в многопользовательском режиме.
**Visual Layout Concept:** Темная тема HSL (`hsl(222, 47%, 11%)`). Иконки Lucide: `Lock`, `ShieldCheck`. Архитектурная схема брокера секретов, изолирующего LLM от хранилища ключей.
* Фундаментальное правило безопасности агентов: учетные данные (API-ключи, пароли) никогда не должны попадать в контекстное окно самой языковой модели.
* Используйте паттерны брокирования секретов (например, через Composio или Managed Agents), чтобы авторизация SaaS-сервисов происходила строго на уровне выполнения (execution layer), вне досягаемости LLM.
* Строго изолируйте рабочие среды клиентов, используя уникальные параметры `.env` и выделенные контейнеры для каждой сессии.

📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Практикум: Чат-агент RAG с HITL — сборка графа поддержки, требующего аппрува человека в Telegram.
**Visual Layout Concept:** Светлая тема HSL (`hsl(210, 40%, 98%)`). Иконки Lucide: `MessageCircle`, `UserCheck`. Практический пайплайн: сообщение в Telegram -> поиск ответа -> остановка на ревью -> подтверждение.
* Соберите систему поддержки клиентов, объединив Telegram, базу знаний RAG и оркестратор на базе LangGraph.
* Настройте графового агента на автоматический поиск решений известных проблем в базе данных компании.
* Для критических или необратимых действий внедрите узел аппрува человеком (Human-In-The-Loop), который ставит выполнение на паузу и отправляет запрос менеджеру в Telegram/Slack перед финальным ответом.

📊 Слайд 7. Блок 7 (Python-разработка): Настройка динамического управления долгосрочной памятью (user-scoped vs thread-scoped) с Mem0/Letta.
**Visual Layout Concept:** Темная тема HSL (`hsl(222, 47%, 11%)`). Иконки Lucide: `HardDrive`, `Server`. Сравнительная таблица жизненного цикла памяти: краткосрочная сессия против постоянных фактов пользователя.
* Качественное управление памятью требует разделения данных на три слоя: потоковая (`thread-scoped`), пользовательская (`user-scoped`) и самоуправляемая (`self-managed`) память.
* Фреймворки вроде Mem0 отлично подходят для сохранения фактов и предпочтений конкретного пользователя между множеством сессий.
* Подобная динамическая память (включая файловую систему) позволяет агенту «вспоминать» глобальный контекст без засорения оперативного окна контекста сырой историей чатов.

📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Использование Checkpointers (PostgresSaver) для автоматического сохранения состояний.
**Visual Layout Concept:** Светлая тема HSL (`hsl(0, 0%, 100%)`). Иконки Lucide: `GitCommit`, `DatabaseZap`. Иллюстрация сохранения JSON-состояния графа после отработки каждого узла.
* В LangGraph механизм checkpointer (например, `PostgresSaver`) передается при компиляции графа и работает как автоматический фиксатор состояния.
* Он сериализует и сохраняет весь объект `State` в базу данных после успешного выполнения каждого узла маршрутизации или инструмента.
* Это критически важно для долгоживущих агентов (long-running agents), чьи задачи могут длиться часами: при сбое инфраструктуры процесс просто загружается с последнего чекпоинта.

---

📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Time Travel (путешествие во времени): отладка графа путем отката состояния и перезаписи.
**Visual Layout Concept:** Темная тема HSL (`hsl(222, 47%, 11%)`). Иконки Lucide: `History`, `Undo2`. Временная шкала с визуализацией "перемотки" (rewind) графа назад к предыдущему состоянию.
* Благодаря сохранению чекпоинтов LangGraph поддерживает уникальную функцию "Time Travel" (путешествие во времени).
* Вы можете в любой момент остановить граф, извлечь конкретное историческое состояние, изменить данные В источниках (например, исправить промпт или результат тула) и запустить выполнение с этой точки.
* Это фундаментальный механизм для отладки галлюцинаций (rewind and fork), позволяющий направлять агента на верный путь без необходимости переделывать всю задачу с нуля.

📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Внедрение концепции Human-in-the-loop (HITL): прерывания (interrupt_before, interrupt_after).
**Visual Layout Concept:** Светлая тема HSL (`hsl(210, 40%, 98%)`). Иконки Lucide: `PauseCircle`, `ShieldAlert`. Блок-схема с красным узлом (шлюзом), требующим внешней авторизации перед продолжением.
* Никогда не доверяйте автономным моделям выполнение деструктивных или финансовых операций (например, изменение БД или отправку писем) без контроля.
* При компиляции графа в LangGraph используйте параметры `interrupt_before` или `interrupt_after` для указания узлов, перед которыми конечный автомат должен заморозить свою работу.
* В этом состоянии `Human-in-the-loop` система ожидает внедрения внешнего сигнала (через API-запрос от оператора), после чего безопасно продолжает выполнение рабочего процесса.
