# Презентация: Введение в мультиагентные системы: CrewAI

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Концепция мультиагентов — разделение ролей, ответственности и наборов инструментов.
* **Visual Layout Concept:** Тёмная тема (Dark mode), неоново-синие акценты. Иконки Lucide: `Users` (Роли), `ListTodo` (Задачи), `Wrench` (Инструменты).
* **Основа Augmented LLM:** Агент — это языковая модель, усиленная доступом к инструментам, памяти и слою оркестрации.
* **Разделение ролей:** Присвоение агентам уникальных персон (например, Planning Agent или Execution Agent) фокусирует их логику и предотвращает галлюцинации.
* **Границы задач:** Принцип принудительного лимита WIP=1 (одна активная задача) предотвращает размытие фокуса и неконтролируемое расширение скоупа (overreach).
* **Изоляция инструментов:** Инструменты служат мостом к внешним API; выдача агенту только необходимых инструментов снижает риск ошибок.

---

📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Запуск по вебхуку — триггеры запуска CrewAI-скриптов по вебхукам.
* **Visual Layout Concept:** Светлая тема (Light mode), схема потока данных (Flowchart). Иконки Lucide: `Webhook`, `Play`.
* **Событийно-ориентированная архитектура:** Вебхуки выступают связующим звеном, позволяя внешним платформам запускать автономные скрипты.
* **Триггеры в n8n:** Узел Webhook захватывает входящие HTTP-запросы и мгновенно инициирует цепочку выполнения (workflow).
* **Передача контекста:** Полезная нагрузка вебхука (JSON payload) безопасно пробрасывает исходные параметры в Python-окружение для старта команды CrewAI.

---

📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Интерфейсы для CrewAI — передача результатов выполнения сложных задач на дашборды.
* **Visual Layout Concept:** Split-screen (разделенный экран: бэкенд и UI дашборд). Иконка Lucide: `LayoutDashboard`.
* **Замыкание цикла:** После выполнения работы агент использует HTTP Request для отправки готового артефакта обратно в систему клиента.
* **UI Дашборды:** Использование фронтенд-интерфейсов для отображения статуса агента (например, "В процессе", "Готово") и финальных данных.
* **Human-in-the-loop (HITL):** Интерфейсы позволяют человеку проверять промежуточные результаты (например, в Slack) перед тем, как агент выполнит необратимое действие.

---

📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Форматирование обмена — подготовка структурированных данных для передачи между агентами.
* **Visual Layout Concept:** Визуализация JSON-схемы поверх кода. Иконка Lucide: `FileJson`.
* **Чистая передача (Clean Handoff):** Правило, обязывающее агента формировать четкий артефакт в конце своей сессии, а не передавать сырые логи.
* **Структурированный вывод:** Использование Pydantic или JSON-парсеров гарантирует, что вывод Агента А идеально матчится со входом Агента Б.
* **Защита от Verification Gap:** Строгие схемы мешают агенту объявить об успехе, если он сгенерировал неполные или битые данные.

---

📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Асинхронные вебхуки — разработка слушателей для параллельного запуска команд.
* **Visual Layout Concept:** Схема сетевой очереди (Network queue). Иконка Lucide: `Network`.
* **Параллелизм:** Асинхронные слушатели позволяют системе принимать множество вебхук-триггеров одновременно, не теряя входящие данные.
* **Управление нагрузкой:** Применение батчинга (Split In Batches) в автоматизациях защищает сервер от перегрузки.
* **Rate Limits:** Очереди выступают буфером, спасая API провайдеров (OpenAI/Anthropic) от ошибок HTTP 429 (Слишком много запросов) при массовых запусках агентов.

---

📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Облачный CrewAI — деплой локальных CrewAI-команд на облачные хостинги.
* **Visual Layout Concept:** Иллюстрация облачной серверной стойки. Иконки Lucide: `Cloud`, `Container`.
* **Вывод в продакшен:** Деплой Python-агентов на платформы вроде Render или Railway для обеспечения работы 24/7.
* **Ловушка эфемерных дисков:** Контейнеры регулярно перезагружаются; локальное сохранение логов ведет к полной потере памяти агента.
* **Durable Execution:** Состояние системы обязательно должно сохраняться во внешних базах данных (PostgreSQL) или в хранилище S3.

---

📊 Слайд 7. Блок 7 (Python-разработка): Создание структуры CrewAI: определение агентов (Agent), задач (Task) и сборка команды (Crew).
* **Visual Layout Concept:** Объектно-ориентированная структура (подсветка синтаксиса Python). Иконка Lucide: `Code`.
* **Класс Agent:** Описывает роль и цель. Критично применять "прогрессивное раскрытие", чтобы избежать раздувания инструкций (Instruction Bloat).
* **Класс Task:** Жестко ограничивает скоуп. Параметр `expected_output` выступает критерием готовности, заставляя LLM проверять свою работу.
* **Оркестрация Crew:** Сборка агентов и задач в конвейер и вызов метода `kickoff()` для старта когнитивного цикла.

---

📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Паттерны мультиагентной координации в производственной среде.
* **Visual Layout Concept:** Топология направленного ациклического графа (DAG). Иконка Lucide: `Workflow`.
* **Workflows vs Agents:** В рабочих процессах поток жестко задан, в агентных системах маршрутизацию определяет сама LLM.
* **Налог на мультиагентность:** Рой агентов решает задачи лучше на 90%, но увеличивает расход токенов контекста до 15 раз.
* **Выбор топологии:** Использование последовательных, иерархических или debate-паттернов зависит от уровня неопределенности бизнес-задачи [32-34].

---

📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Последовательный vs Иерархический процессы управления с кастомным Supervisor Pattern.
* **Visual Layout Concept:** Топология "Звезда" с центральным узлом. Иконка Lucide: `GitMerge`.
* **Последовательный процесс:** Строгий, детерминированный конвейер, идеальный для повторяющихся задач (например, парсинг данных).
* **Иерархический процесс:** Агент-Супервизор (Оркестратор) самостоятельно декомпозирует задачу и распределяет работу между агентами-воркерами.
* **Оптимизация 60/30/10:** Использование мощных моделей (Opus/GPT-4o) для Супервизора и дешевых (Haiku/Mini) для линейных воркеров.

---

📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Архитектура общей памяти (Shared Memory) между независимыми агентами.
* **Visual Layout Concept:** Схема базы данных, соединенной с графом знаний. Иконка Lucide: `Database`.
* **Мастер с амнезией:** Без сохранения контекста каждая сессия сжигает токены, заново изучая базовые правила проекта.
* **Файловые регистры:** Использование файлов Markdown (``, `wiki/`) как долговечного источника истины для всех агентов в рое (Метод Карпатого).
* **Контекст-Инжиниринг:** Агенты сохраняют длинные логи в БД, а извлекают только сжатые векторы знаний, избегая переполнения контекстного окна.
