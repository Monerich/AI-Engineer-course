# Презентация: Кейс-проект: Локальный CLI Агент с инструментами

📊 **Slide 1. Блок 1 (ИИ-Инженер / Автоматизация): Интегрированный CLI-инструментарий**
* **Visual Layout Concept:** Тёмный режим (HSL 220, 15%, 15%), иконки Lucide: `Terminal`, `Hexagon`.
* **Key bullet points:**
 * Сборка базовых инструментов для локального агента: `web_search` (Tavily/Firecrawl), `read_file` и `write_file`.
 * Построение основного цикла: запрос модели → диспетчеризация инструмента → возврат результата модели.
 * Описания параметров и инструментов служат строгими инструкциями для LLM, чтобы модель выбирала их безошибочно.

---
📊 **Slide 2. Блок 2 (ИИ-Инженер / Автоматизация): Сессионное хранение**
* **Visual Layout Concept:** Светлый режим (HSL 210, 20%, 95%), иконки Lucide: `Database`, `Save`. Схема взаимодействия с SQLite.
* **Key bullet points:**
 * Реализация «Durable resume» (надежного восстановления) с использованием локальной БД SQLite для сохранения истории.
 * Каждый шаг агента фиксируется транзакционно для защиты от случайных прерываний (Ctrl+C).
 * Интеграция метода Карпатого: использование файловой системы и `` как долговременной памяти для «инженера с амнезией».

---
📊 **Slide 3. Блок 3 (ИИ-Инженер / Автоматизация): Логирование выполнения**
* **Visual Layout Concept:** Тёмный режим (HSL 200, 10%, 20%), иконки Lucide: `List`, `Eye`.
* **Key bullet points:**
 * Внедрение рантайм-наблюдаемости (observability) для визуализации скрытого цикла ReAct.
 * Визуальное разделение когнитивной фазы (внутренний монолог), вызова инструмента и результата физического мира.
 * Сбор телеметрии (OTEL-spans, токены, задержка) для каждого шага агента для последующего анализа.

---
📊 **Slide 4. Блок 4 (ИИ-Инженер / Автоматизация): Мониторинг директорий**
* **Visual Layout Concept:** Светлый режим (HSL 150, 15%, 90%), иконки Lucide: `FolderSearch`, `Activity`.
* **Key bullet points:**
 * Переход от реактивных к фоновым (ambient) агентам, которые просыпаются по событиям системы.
 * Написание Python-скриптов (File Watcher) для перехвата обновлений в файловой системе.
 * Использование паттерна Debounce для фильтрации «дребезга» ОС и предотвращения сжигания API-бюджета.

---
📊 **Slide 5. Блок 5 (ИИ-Инженер / Автоматизация): Автодокументирование**
* **Visual Layout Concept:** Тёмный режим (HSL 280, 20%, 15%), иконки Lucide: `FileText`, `PenTool`.
* **Key bullet points:**
 * Внедрение парадигмы «Чистой передачи» (Clean Handoff): каждая сессия обязана оставлять чистое состояние перед закрытием.
 * Автоматическая генерация отчетов и обновление файла `` перед завершением процесса.
 * Использование изолированного Writer-агента для компиляции итогов на основе сырых логов SQLite.

---
📊 **Slide 6. Блок 6 (ИИ-Инженер / Автоматизация): Практикум: Сборка CLI-агента**
* **Visual Layout Concept:** Светлый режим (HSL 45, 20%, 95%), иконки Lucide: `Bot`, `Wrench`. Диаграмма общего цикла агента.
* **Key bullet points:**
 * Интеграция поисковых и файловых (read, write, grep, glob) инструментов в единого ассистента.
 * Сборка собственной мини-обвязки (harness) в ~1500 строк Python без использования тяжелых фреймворков.
 * Реализация загрузчика системного промпта из `` и реестра инструментов.

📊 **Slide 7. Блок 7 (Python-разработка): Использование библиотеки Rich**
* **Visual Layout Concept:** Тёмный режим (HSL 250, 15%, 15%), иконки Lucide: `Palette`, `LayoutTemplate`.
* **Key bullet points:**
 * Переход от сырых логов (ANSI) к премиальным визуальным CLI-макетам с библиотекой `rich`.
 * Использование `rich.Status` и `rich.Live` для отображения динамических спиннеров при долгих вызовах инструментов.
 * Рендеринг размышлений агента в Markdown-панелях и подсветка синтаксиса JSON для аргументов.

📊 **Slide 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Скрытие внутренних размышлений**
* **Visual Layout Concept:** Светлый режим (HSL 0, 0%, 98%), иконки Lucide: `EyeOff`, `MessageSquare`.
* **Key bullet points:**
 * Использование XML-тегов (например, `<thought>`) для создания отдельного канала внутреннего монолога (Precognition).
 * Реализация потокового парсера (Streaming State Machine) для фильтрации черновика от пользователя в рантайме.
 * Сохранение скрытых мыслей в базу данных SQLite для отладки, выводя клиенту только отполированный результат.

---
📊 **Slide 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Алгоритмы сжатия и очистки контекстного окна**
* **Visual Layout Concept:** Тёмный режим (HSL 0, 10%, 20%), иконки Lucide: `Minimize`, `HardDrive`.
* **Key bullet points:**
 * Контроль лимитов контекста (Context Compaction) для предотвращения ошибки `400 Token Limit Exceeded`.
 * Паттерн Filesystem Offload: автоматическая выгрузка тяжелых результатов (>20K токенов) на диск с возвратом превью.
 * Суммаризация истории при заполнении 85% окна с явным сохранением архитектурных ограничений ("Почему" это было сделано).

📊 **Slide 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Сохранение, загрузка сессий и восстановление состояния**
* **Visual Layout Concept:** Светлый режим (HSL 200, 20%, 95%), иконки Lucide: `RefreshCw`, `ShieldCheck`.

 * Стратегия «теплого старта»: загрузка Bootstrap-контракта (`` и ``) перед каждой новой сессией.
