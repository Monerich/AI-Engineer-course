# Презентация: Деплой и Надежное Выполнение

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Docker в ИИ-разработке — создание оптимизированных Docker-файлов для упаковки систем.
**Visual Layout Concept:** Темный фон (Slate 900), акцент на контрастных HSL-блоках кода, иконка Lucide `Container`.
* Упаковка кода, зависимостей и ИИ-моделей в единый изолированный контейнер.
* Гарантирует, что агент будет работать идентично на локальном ноутбуке и на облачном сервере.
* Является фундаментальной базой для развертывания веб-сервисов и работы с базами данных.

---
📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Docker Compose — конфигурирование Docker Compose сетей для взаимодействия ИИ-сервисов.
**Visual Layout Concept:** Разделенный экран (код конфигурации vs схема сети), иконка Lucide `Network`.
* Оркестрация нескольких связанных контейнеров, таких как AI-фреймворк (n8n), PostgreSQL и брокер очередей.
* Настройка безопасной локальной сетевой маршрутизации между микросервисами.
* Позволяет быстро и безопасно тестировать полностековые архитектуры перед выходом в продакшен.

---
📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Облачный деплой — развертывание контейнеров на Railway, Render, AWS, GCP.
**Visual Layout Concept:** Сетка из четырех колонок с логотипами провайдеров (светлая тема), иконка Lucide `CloudUpload`.
* Использование Railway или Render для мгновенного развертывания прототипов с поддержкой HTTPS и пользовательских доменов.
* Применение AWS и GCP для глобального масштабирования решений, обслуживающих тысячи пользователей.
* Строгое управление секретами: API-ключи внедряются только через переменные окружения, никогда в коде.

---
📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Автоскейлинг — настройка систем горизонтального автоскейлинга под нагрузкой.
**Visual Layout Concept:** Динамический линейный график (нагрузка vs воркеры), иконка Lucide `TrendingUp`.
* Горизонтальное масштабирование stateless-воркеров для обработки резких всплесков веб-трафика.
* Использование режима очередей (queue mode) для буферизации задач и защиты серверов от перегрузок.
* Интеграция CI/CD пайплайнов для автоматического выпуска обновлений и расширения инфраструктуры.

---
📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Балансировка и кэш — развертывание балансировщиков нагрузки и кэширования запросов к LLM.
**Visual Layout Concept:** Схема маршрутизации трафика к кэш-слою, иконка Lucide `DatabaseZap`.
* Агрессивное кэширование системных промптов и инструментов экономит до 90% затрат на API.
* Маршрутизация по сложности: простые задачи уходят быстрым моделям, сложное планирование — флагманам.
* Использование проксирования и локального кэша для снижения задержек и обхода лимитов (Rate Limits).

---
📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): High Availability — проектирование архитектуры высокой доступности для корпоративных ИИ-решений.
**Visual Layout Concept:** Схема отказоустойчивости с резервными узлами (красные/зеленые индикаторы), иконка Lucide `ShieldCheck`.
* Проектирование архитектуры, устойчивой к сбоям облачного провайдера или отключениям LLM API.
* Внедрение circuit-breaker и автоматического отката (kill switch) при деградации метрик.
* Непрерывный мониторинг производительности для отлова скрытых регрессий и "дрейфа" промптов.

---
📊 Слайд 7. Блок 7 (Python-разработка): Разработка высокопроизводительных асинхронных веб-сервисов для ИИ-приложений на FastAPI.
**Visual Layout Concept:** Блок кода с подсветкой `async def`, иконка Lucide `Terminal`.
* Создание масштабируемых бэкендов на FastAPI с использованием Pydantic для строгой валидации данных.
* Применение асинхронных (async) функций, чтобы долгие вызовы LLM не блокировали основной поток сервера.
* Прямая интеграция FastAPI-сервиса с базами данных (PostgreSQL) для надежного сохранения состояния.

---
📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Концепция надежного выполнения долгоживущих процессов (Durable Execution) при сбоях.
**Visual Layout Concept:** Блок-схема чекпоинтов базы данных, иконка Lucide `SaveAll`.
* Durable execution необсуждаем для любого агента, работающего дольше 60 секунд.
* Сохранение состояния агента после каждого выполненного узла (например, через PostgresSaver), делая сбои обратимыми.
* Мгновенное восстановление прерванных сессий из базы данных после падения контейнера по нехватке памяти (OOM).

---
📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Интеграция систем автоматического оркестрирования шагов выполнения (Temporal / Inngest SDK).
**Visual Layout Concept:** Двухуровневая архитектура (Workflows vs Activities), иконка Lucide `Workflow`.
* Интеграция Inngest или Temporal SDK превращает каждый вызов инструмента агентом в надежный шаг (durable step).
* Строгое разделение детерминированных рабочих процессов (workflows) и хаотичной логики взаимодействия с внешними API (activities).
* Платформа автоматически обрабатывает таймауты, экспоненциальные задержки и повторные попытки при падении LLM API.

---
📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Обеспечение идемпотентности шагов графа рассуждений агента при аварийных перезапусках.
**Visual Layout Concept:** Схема логических вентилей с блокировками дублирования, иконка Lucide `RefreshCw`.
* Внедрение ключей идемпотентности для каждого шага предотвращает двойное списание средств или дублирование писем при ретраях.
* Идемпотентная очистка гарантирует, что удаление временных файлов безопасно даже в сценариях «сбой-ретрай».
* Защита графа рассуждений: безопасный перезапуск in-flight вызовов без повреждения внешних сервисов.
