# Презентация: Контекст-инжиниринг и современные методы промптинга

**Визуальный концепт:** Тёмная тема (HSL: 220, 15%, 15%). Иконки Lucide: `Database`, `Cpu`.
**Ключевые тезисы:**
* В 2026 году классический prompt engineering уступил место context engineering.

* Эффективная работа требует жесткого контроля за Instruction Bloat и переполнением контекста.

📊 Слайд 2. Блок 2: Системные инструкции (System Prompts) для жесткого управления форматом и структурой ответа
**Визуальный концепт:** Светлая тема (HSL: 210, 20%, 98%). Иконки Lucide: `TerminalSquare`, `Settings`.
**Ключевые тезисы:**
* Системные промпты задают глобальную цель, роль и ограничения для языковой модели.
* Они критически важны для принудительного возврата структурированного вывода (например, строгого JSON).
* Жестко заданный формат вывода — основа интеграции ИИ в автоматизированные рабочие процессы.

📊 Слайд 3. Блок 3: XML-теговый промптинг — шаблонизация промптов и разграничение контекста для защиты от инъекций
**Визуальный концепт:** Тёмная тема (HSL: 200, 50%, 15%). Иконки Lucide: `Code`, `ShieldCheck`.
**Ключевые тезисы:**
* Использование XML-тегов позволяет четко отделить системные инструкции от пользовательских данных.
* Запрет на использование угловых скобок (`< >`) во внутренних метаданных предотвращает внедрение вредоносных инструкций.
* Теги улучшают способность модели следовать сложным многоуровневым правилам.

📊 Слайд 4. Блок 4: Снижение уровня галлюцинаций LLM — методы верификации фактов и обработка неопределенности
**Визуальный концепт:** Светлая тема (HSL: 0, 0%, 100%). Иконки Lucide: `CheckCircle`, `AlertTriangle`.
**Ключевые тезисы:**
* Для защиты от галлюцинаций модель прямо инструктируют отвечать «Unsure about answer», если фактов в контексте недостаточно.
* Запрос промежуточных рассуждений перед ответом значительно снижает процент выдуманных фактов.
* Агенты не должны объявлять об успехе без независимой верификации.

📊 Слайд 5. Блок 5: Практикум: Сборщик JSON из сырого текста — написание промпта извлечения заказов из писем
**Визуальный концепт:** Тёмная тема (HSL: 220, 10%, 10%). Иконки Lucide: `FileJson`, `Filter`.
**Ключевые тезисы:**

* Эффективный пайплайн требует предоставления модели четкого примера целевого массива JSON.
* В системном промпте жестко прописываются ключи JSON, чтобы избежать галлюцинаций структуры.

📊 Слайд 6. Блок 6: Безопасность контекста — проектирование защитных инструкций (Prompt Guard) против утечки промптов
**Визуальный концепт:** Светлая тема (HSL: 0, 0%, 95%). Иконки Lucide: `Lock`, `ShieldAlert`.
**Ключевые тезисы:**

* Изоляция среды и фильтрация ввода защищают агента от перехвата управления.
* Архитектура должна разделять «того, кто выполняет задачу» и «того, кто верифицирует безопасность».

📊 Слайд 8. Блок 8: 4 примитива контекст-инжиниринга: Write (запись), Select (выборка), Compress (сжатие), Isolate (изоляция)

📊 Слайд 9. Блок 9: Few-shot промптинг с Example Selection • Chain of Thought (CoT) с тегами `<thought>`
**Визуальный концепт:** Тёмная тема (HSL: 220, 15%, 15%). Иконки Lucide: `ListPlus`, `BrainCircuit`.
**Ключевые тезисы:**
* Few-shot промптинг использует блоки `<example>`, показывая модели эталонный формат ответа.
* Chain of Thought (CoT) заставляет LLM генерировать промежуточные шаги рассуждений, что повышает точность сложной логики.
* Теги `<thought>` скрывают логику модели от пользователя, выдавая только чистый финальный результат.

📊 Слайд 10. Блок 10: Концептуальный разбор и логика цикла рассуждений ReAct (Reason-Act)
**Визуальный концепт:** Светлая тема (HSL: 0, 0%, 100%). Иконки Lucide: `RefreshCw`, `Bot`.
**Ключевые тезисы:**
* Фреймворк ReAct объединяет рассуждение и вызов инструментов в единый цикл.
* Агент генерирует мысль (Thought), вызывает инструмент (Action) и получает данные из среды (Observation).
* Цикл повторяется до тех пор, пока агент не соберет достаточно контекста для финального ответа.
