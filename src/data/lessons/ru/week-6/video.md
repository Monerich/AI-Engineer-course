# Сценарий видео: ИИ-ноды n8n и интеграция с LangChain

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Scene 1:** Крупный план спикера, переход к тёмному интерфейсу n8n. На экране появляется надпись "Advanced AI = LangChain". | Добро пожаловать на шестую неделю. Интеграция Advanced AI в n8n работает поверх мощной библиотеки LangChain. Архитектура модульная: корневые узлы, такие как AI Agent, соединяются с суб-узлами — языковыми моделями, памятью и парсерами. Это позволяет визуально собирать сложные когнитивные пайплайны, четко разделяя логику и инструменты. |
| **Scene 2:** Экран записи интерфейса n8n. Добавляется узел "AI Agent", выбирается модель и подключается триггер чата. | Базовая нода AI Agent может работать в режиме Conversational Agent или ReAct. Агент самостоятельно обрабатывает входящие запросы, управляет диалогом и решает, когда нужно обратиться к внешним источникам данных. В системном промпте мы просто задаём его роль, ограничения и правила использования подключенных инструментов. |
| **Scene 3:** Разделённый экран. Слева — узел HTTP Request в n8n, справа — TypeScript-код кастомной ноды с методом `execute()`. | Для специфических задач базовых инструментов не хватает. Мы используем ноды Custom Code Tool или HTTP Request. Если нужно написать свою ноду для сообщества, логика и формирование JSON описываются в TypeScript-классе через метод execute. При этом авторизация безопасно передаётся через встроенную систему Credentials,. |
| **Scene 4:** Схема работы памяти. Сравнивается "Window Buffer Memory" (временная) и "Postgres Chat Memory" (постоянная база данных). | Память критически важна. Window Buffer Memory хранит контекст только в рамках текущей сессии в оперативной памяти,. Но для долгосрочных агентов это не подходит. Мы подключаем внешние базы данных через Postgres или Redis Chat Memory. Сохранение контекста в БД решает проблему "амнезии" модели при перезапусках. |
| **Scene 5:** Иконки PDF, CSV и TXT файлов залетают в ноду "Default Data Loader" и превращаются во фрагменты текста (JSON). | Чтобы агент отвечал на вопросы по корпоративным базам знаний, файлы загружаются через Document Loaders. В n8n они работают с бинарными данными, преобразуя неструктурированные PDF или структурированные CSV в унифицированные документы. Загрузчики извлекают текст и метаданные, подготавливая их к векторизации. |
| **Scene 6:** Анимация разрезания длинного текста на пересекающиеся блоки (chunks) с помощью иконки ножниц. | Загрузка целого документа в LLM вызывает "раздувание инструкций" и потерю информации в середине текста. Мы используем Text Splitters — например, Recursive Character или Token Splitter. Они математически разбивают текст на чанки с заданным перекрытием, сохраняя семантику на стыках фрагментов. |
| **Scene 7:** Терминал с Python-кодом, использующим официальные SDK OpenAI и Anthropic. Выделены блоки `try/except` и импорт `dotenv`. | Иногда визуальной среды недостаточно. Профессионалы используют официальные Python SDK OpenAI и Anthropic для гранулярного контроля над агентами. Через скрипты мы безопасно управляем переменными окружения, жестко задаем таймауты и обрабатываем сетевые ошибки через try/except, предотвращая зависание системы,. |
| **Scene 8:** Интерфейс настройки инструмента (Tool) в n8n, крупный план поля "Description" и строгой JSON-схемы входных параметров. | Названия инструментов и их описания — это прямые инструкции для языковой модели. Хорошее описание детально объясняет LLM, что инструмент делает и когда его применять. Точная настройка JSON-схем параметров предотвращает галлюцинации агента и гарантирует вызов нужного API с правильными данными. |
| **Scene 9:** Красный экран с предупреждением о "Prompt Injection". Показывается узел проверки перед выполнением (PreToolUse хук). | Автономность несёт риски. Вредоносный промпт может заставить агента выполнить деструктивную команду. Мы внедряем защиту: никогда не храним ключи в коде, санитайзим ввод и используем хуки PreToolUse для блокировки опасных параметров,. Для любых необратимых действий всегда ставьте подтверждение человеком. |
| **Scene 10:** Дашборд платформы LangSmith, показывающий дерево вызовов (OTEL spans) с миллисекундами и токенами. | Без трассировки рассуждений отладка агента превращается в угадывание. Используя платформы вроде LangSmith, мы собираем метрики по каждому вызову модели и инструмента. Мы отслеживаем задержки, потребление токенов и формируем диагностическую петлю для хирургического исправления логических ошибок агента,. |

Это завершает обзор скриптов для шестой недели. Готовы ли вы перейти к детальному разбору седьмой недели, или вас интересует практическая реализация какого-то конкретного блока?
