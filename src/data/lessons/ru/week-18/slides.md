# Презентация: Наблюдаемость и Трейсинг в LangSmith

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Трейсинг OpenTelemetry
**Visual Layout Concept:** Dark mode (HSL: 210, 20%, 15%), схема сквозной трассировки от узла к узлу. Lucide icon: `Activity`.
* OpenTelemetry предоставляет стандартизированный фреймворк для трассировки ИИ-приложений, независимый от вендора.
* Он обеспечивает видимость каждого шага выполнения (span), позволяя отслеживать вызовы LLM, выполнение инструментов и маршрутизацию агентов.
* Интеграция OTEL жизненно важна для выявления инфраструктурных ошибок до того, как они достигнут пользователя.

---
📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Экспорт логов n8n
**Visual Layout Concept:** Light mode (HSL: 0, 0%, 98%), пайплайн выгрузки данных во внешнюю базу. Lucide icon: `DatabaseExport`.
* n8n поддерживает продвинутые механизмы логирования и мониторинга для каждого запущенного воркфлоу.
* Данные выполнения могут быть перенаправлены во внешние хранилища и системы аналитики для долговременного хранения.
* Настройка экспорта критична для аудита ошибок и анализа исторических сбоев в продакшене.

---
📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): LangSmith / Langfuse
**Visual Layout Concept:** Dark mode (HSL: 240, 15%, 12%), иерархический граф (DAG) шагов агента. Lucide icon: `Eye`.
* Платформы вроде LangSmith обеспечивают нативный трейсинг для ИИ-агентов, делая каждый шаг их рассуждений видимым.
* Вы можете просматривать точные промпты, ответы модели и использование инструментов в каждом узле.
* Трейсинг позволяет проводить точечную отладку и внедрять online/offline оценки (evals) качества работы.

---
📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Финансовый мониторинг
**Visual Layout Concept:** Light mode (HSL: 120, 20%, 95%), столбчатая диаграмма с подсчетом токенов. Lucide icon: `DollarSign`.
* Мульти-агентные системы могут потреблять до 15 раз больше токенов, чем базовые запросы, требуя строгой финансовой дисциплины.
* Мониторинг должен отслеживать метрику cost-per-task и общие затраты на вызовы API.
* Эффективный трейсинг позволяет увидеть, где кэширование промптов может сэкономить бюджет.

---
📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Мониторинг задержек
**Visual Layout Concept:** Dark mode (HSL: 0, 0%, 10%), линейный график времени отклика. Lucide icon: `Timer`.
* Мониторинг критических метрик вроде p95 latency помогает выявить узкие места в архитектуре.
* Время до первого токена (TTFT) и общая задержка (latency) снижаются за счет стриминга и параллельных вызовов инструментов.
* Трассировка отличает задержки самой модели от инфраструктурного шума и медленных API-ответов сторонних сервисов.

---
📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Лимиты и оповещения
**Visual Layout Concept:** Light mode (HSL: 0, 80%, 96%), всплывающие уведомления (алерты). Lucide icon: `BellRing`.
* Системы мониторинга должны включать алерты на превышение стоимости, рост доли отказов (failure rate) и деградацию задержек.
* Уведомления об ошибках и лимитах автоматически маршрутизируются в Telegram или Slack для немедленного реагирования.
* Грамотные алерты предотвращают ситуации, когда циклический сбой агента незаметно сжигает бюджет.

---
📊 Слайд 7. Блок 7 (Python-разработка): Интеграция SDK LangSmith/Langfuse
**Visual Layout Concept:** Dark mode (HSL: 220, 10%, 15%), фрагмент Python-кода с декораторами. Lucide icon: `Code`.
* SDK платформ наблюдаемости интегрируются на уровне Python-функций, оборачивая логику в иерархические спаны (spans).
* Это критически важно для трассировки "холодных стартов" — фазы инициализации контекста до вызова LLM.
* Использование декораторов позволяет динамически тегировать сессии и передавать метаданные в дашборды.

---
📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов): Harness Engineering. Лекция 1
**Visual Layout Concept:** Light mode (HSL: 45, 50%, 96%), пятислойный защитный щит. Lucide icon: `ShieldAlert`.
* Сильная модель (например, Opus 4.5) не гарантирует надежного результата без правильной обвязки (harness).
* Отказы часто происходят из-за Verification Gap, когда агент преждевременно заявляет об успехе.
* При сбоях сначала нужно чинить инфраструктуру обвязки (Definition of Done, контекст), а не винить или менять саму модель.

---
📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов): Harness Engineering. Лекция 2
**Visual Layout Concept:** Dark mode (HSL: 260, 15%, 15%), изолированный контейнер-песочница. Lucide icon: `Box`.
* Тестовый стенд (harness) состоит из 5 подсистем: инструкции, инструменты, рантайм (изоляция), обратная связь и состояние.
* Среда исполнения помещает агента в песочницу, не давая ему сломать реальную систему.
* Подсистема обратной связи (симуляция ввода) перехватывает логи тестов и автоматически возвращает их агенту для исправления ошибок.

---
📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов): Harness Engineering. Лекции 3 и 4
**Visual Layout Concept:** Light mode (HSL: 210, 40%, 96%), структура директорий с документами. Lucide icon: `FolderTree`.
* Репозиторий кода должен быть единственным источником правды; знания вне репозитория для агента не существуют.
* Избегайте "Instruction Bloat": огромные файлы промптов перегружают контекст и вызывают эффект "Lost in the Middle".
* Инструкции нужно разделять по файлам и подгружать динамически (Progressive Disclosure) в зависимости от задачи агента.
