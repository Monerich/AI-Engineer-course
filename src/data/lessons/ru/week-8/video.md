# Сценарий видео: Кейс-проект: Автопилот лидогенерации и холодного аутрича

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Сцена 1:** Спикер в студии. На экране логотипы Apollo.io и Apify, соединенные стрелками, формирующие сырой массив данных. | Добро пожаловать на восьмую неделю. Сегодня мы строим автопилот лидогенерации. Всё начинается со скрейпинга. Прямой экспорт из баз данных часто дорог, поэтому мы используем сторонние скрейперы вроде Apify для сбора профилей из Apollo. На выходе мы получаем сырой массив данных, готовый к обработке. |
| **Сцена 2:** Схема профиля компании обогащается новыми блоками данных: "Стек технологий", "Вакансии". Логотип Clay. | Сырые лиды содержат пробелы. На втором этапе мы используем пайплайны обогащения данных через инструменты вроде Clay. ИИ анализирует страницы «О нас» и свежие вакансии компании. Если они активно ищут DevOps-инженеров, значит, их инфраструктура масштабируется. Мы собираем реальные бизнес-сигналы. |
| **Сцена 3:** Анимация генерации текста. На экране появляется JSON-объект с полями `subject`, `icebreaker`, `elevator_pitch`. | Шаблонные холодные письма мертвы. Мы переходим к ИИ-персонализации. Агент изучает профиль лида в LinkedIn и генерирует первую фразу — icebreaker, которая имитирует глубокий ручной ресерч. Мы жестко просим модель возвращать результат в формате JSON для дальнейшего парсинга. |
| **Сцена 4:** Нода n8n передает данные в интерфейс HubSpot. Создается новая карточка сделки (Deal). | Лиды с высоким рейтингом не должны висеть в таблицах. Мы настраиваем автосоздание сделок и контактов в CRM-системах, таких как HubSpot или Pipedrive, через API. Это устраняет ручной ввод данных и экономит отделу продаж десятки часов в неделю. |
| **Сцена 5:** Логотипы Instantly и Smartlead. График отправки писем, показывающий соблюдение лимитов домена. | Мы никогда не делаем массовые email-рассылки напрямую из n8n. Сгенерированные письма передаются по API в платформы вроде Instantly или Smartlead. Это защищает репутацию ваших доменов и позволяет управлять лимитами отправки автоматически. |
| **Сцена 6:** Скринкаст большого сквозного воркфлоу n8n. Курсор выделяет узел `Loop` (Split In Batches). | Давайте соберем эту аутрич-машину воедино. Сквозной n8n-процесс должен быть надежным. Чтобы избежать переполнения памяти при обработке тысяч лидов, мы используем узлы порционной обработки — Loop. Система последовательно берет небольшие батчи, персонализирует их и отправляет дальше. |
| **Сцена 7:** Окно узла `Code` в n8n. Пишется короткий скрипт на Python для фильтрации словарей (Dicts). | Выгрузки из скрейперов часто содержат мусор: пустые строки или битые кодировки. Перед отправкой данных в LLM мы используем Python для фильтрации и нормализации CSV-выгрузок. Отсеивая невалидные записи на уровне кода, мы экономим токены и предотвращаем сбои API. |
| **Сцена 8:** Файл `` открывается на экране. Красным зачеркиваются клише вроде "Ваш надежный партнер". | По умолчанию ИИ звучит роботизированно. Чтобы этого избежать, мы загружаем файл с примерами нашего стиля. Промпт жестко задает «лаконичный спартанский тон» и запрещает ИИ использовать маркетинговый буллшит или корпоративные клише. |
| **Сцена 9:** Узел `Switch` в n8n разделяет поток писем на ветки: `INTERESTED`, `NOT NOW`, `SPAM`. | Аутрич — это двусторонний процесс. Когда лид отвечает, легковесная модель мгновенно классифицирует тональность ответа. Если это отписка, контакт убирается из базы. Если лид заинтересован, система автоматически маршрутизирует сообщение и тегает нужного менеджера продаж в Slack. |
| **Сцена 10:** Дашборд юнит-экономики. Сравнение стоимости вызовов GPT-4o-mini и Claude 3.5 Sonnet. | И, наконец, экономика токенов. Вы не можете отправлять каждую задачу в самую дорогую модель. Мы используем маршрутизацию: дешевые модели для сортировки, дорогие — для креатива. Агрессивное кэширование промптов снижает расходы до 90%. Так ваш пайплайн с абонентской платой в $1000 приносит реальную прибыль. |
