# Презентация: Кейс-проект: Автопилот лидогенерации и холодного аутрича

📊 Слайд 1. Блок 1: Скрейпинг лидов — сбор базы лидов с помощью скрейперов Apollo и Clay
**Визуальный концепт:** Тёмная тема (HSL: 220, 15%, 15%). Иконки Lucide: `Database`, `Search`. Визуализация воронки скрейпинга от исходного URL до массива сырых данных.
**Ключевые тезисы:**
* Сбор данных из Apollo.io чаще всего автоматизируется через сторонние скрейперы (например, Apify), чтобы обойти дорогие прямые лимиты [1-3].
* Система принимает параметры поиска (должность, индустрия, локация) и извлекает сотни профилей за запуск.
* На выходе формируется сырой массив (JSON или CSV), содержащий имена, названия компаний и ссылки на социальные сети, готовый к дальнейшей обработке.

📊 Слайд 2. Блок 2: Обогащение данных — пайплайны обогащения B2B-контактов по социальным сетям и доменам
**Визуальный концепт:** Светлая тема (HSL: 210, 20%, 98%). Иконки Lucide: `Cpu`, `Globe`. Схема перехода от пустых профилей к заполненным карточкам.
**Ключевые тезисы:**
* Сырые лиды содержат пробелы. Обогащение (через инструменты вроде Clay) добавляет контекст: размер команды, используемый стек технологий и свежие новости.
* ИИ анализирует страницы «О нас» и раздел вакансий, чтобы вычислить реальные бизнес-приоритеты (например, если ищут DevOps — компания масштабируется).
* Этот шаг фильтрует нецелевые контакты, оставляя только тех, кто строго соответствует профилю идеального клиента (ICP).

📊 Слайд 3. Блок 3: ИИ-персонализация — генерация писем-оупенеров на основе профилей LinkedIn лидов
**Визуальный концепт:** Тёмная тема с градиентами (HSL: 200, 50%, 15%). Иконки Lucide: `MessageSquare`, `Sparkles`. Демонстрация JSON-вывода с icebreaker'ами.
**Ключевые тезисы:**
* Шаблонные рассылки с простыми переменными мертвы; современный аутрич требует генерации многострочных "icebreaker" оупенеров.
* ИИ читает обогащенные данные сайта и LinkedIn профиля, чтобы создать уникальную первую фразу, имитирующую глубокий ручной ресерч.
* Модель получает строгий промпт для вывода данных в формате JSON, разделяя тему письма, оупенер, элеватор-питч и призыв к действию [12-14].

**Визуальный концепт:** Светлая тема (HSL: 0, 0%, 100%). Иконки Lucide: `Briefcase`, `ArrowRightLeft`. Блок-схема маршрутизации данных в CRM.
**Ключевые тезисы:**

* Это устраняет ручной ввод данных менеджерами, экономя отделу продаж до 10 часов в неделю.
* Привязка API позволяет сразу назначать задачи (фоллоу-апы) нужным сотрудникам.

📊 Слайд 5. Блок 5: Массовые рассылки — интеграция и запуск цепочек писем через Instantly/Smartlead
**Визуальный концепт:** Тёмная тема (HSL: 220, 10%, 10%). Иконки Lucide: `Mail`, `Send`. Схема передачи готового JSON в платформу рассылки.
**Ключевые тезисы:**
* Сгенерированные ИИ письма не отправляются напрямую из n8n, а передаются в специализированные платформы почтовой инфраструктуры, такие как Instantly.ai или Smartlead.
* Интеграция позволяет автоматически добавлять лидов в активные кампании (one-click campaigns) через API.
* Такой подход защищает репутацию доменов и позволяет управлять лимитами отправки.

📊 Слайд 6. Блок 6: Практикум: Сборка аутрич-машины — сквозной n8n-процесс холодного ИИ-аутрича
**Визуальный концепт:** Сплит-скрин. Иконки Lucide: `Settings`, `Workflow`. Визуализация всей цепочки n8n на одном экране.
**Ключевые тезисы:**
* Сквозной пайплайн объединяет все шаги: триггер из базы -> скрейпинг -> агрегация данных -> ИИ-персонализация -> отправка.
* Используются узлы Split in Batches и Loop для последовательной, порционной обработки массивов лидов без перегрузки памяти.
* Внедряется базовая логика фильтрации (например, удаление дубликатов), чтобы дважды не писать одному и тому же лиду.

**Визуальный концепт:** Тема редактора кода (Тёмная). Иконки Lucide: `FileCode2`, `Filter`. Кусок Python кода для работы с Pandas/Словарями.

📊 Слайд 9. Блок 9: Классификация ответов лидов по тональности (Заинтересован, Спам, Отписка)
**Визуальный концепт:** Тёмная тема. Иконки Lucide: `Tags`, `GitMerge`. Маршрутизация (Switch node) на 4 ветки.
**Ключевые тезисы:**
* Когда клиент отвечает, легковесная модель (например, GPT-4o-mini) анализирует текст для определения намерения.
* Ответы жестко классифицируются в JSON-теги: INTERESTED, NOT NOW, UNSUBSCRIBE, QUESTION.
* Узел Switch направляет ответ по нужной ветке: отписка добавляется в блек-лист, а заинтересованный лид мгновенно отправляется менеджеру в Slack/CRM.

📊 Слайд 10. Блок 10: Экономика токенов в массовых рассылках: расчет ROI ИИ-системы
**Визуальный концепт:** Светлая тема. Иконки Lucide: `TrendingUp`, `Coins`. Дашборд с расчетом юнит-экономики COGS.
**Ключевые тезисы:**
* Массовые операции требуют Дисциплины стоимости: маршрутизация 60/30/10 гарантирует, что 60% простых задач (сортировка) идут в дешевые модели, а дорогие используются только для креатива.
* Использование Prompt Caching и Batch API может снизить расходы на токены до 90% для фоновых задач генерации.
* Строгий расчет стоимости на один лид (Unit Economics) позволяет уверенно продавать ИИ-пайплайны клиентам за $1000+/мес при копеечных затратах на API.
