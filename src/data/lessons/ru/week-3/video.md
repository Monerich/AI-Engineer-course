# Сценарий видео: Основы n8n и первый рабочий процесс

| Кадр / Scene | Звуковое сопровождение / Voiceover |
|:--- |:--- |
| **Сцена 1: Ведущий в кадре, на фоне анимированная схема развертывания Docker Compose + PostgreSQL.** | Добро пожаловать на третью неделю. Мы начинаем с фундамента — инфраструктуры n8n. Для серьезных задач стандарт индустрии — это self-hosted развертывание через Docker в связке с базой данных PostgreSQL. Это дает вам абсолютный контроль над данными и снимает любые облачные лимиты на количество выполнений. |
| **Сцена 2: Запись экрана интерфейса n8n, фокус на панели добавления триггеров и узлах действий.** | Любой рабочий процесс в n8n начинается с триггера на визуальном холсте. Они могут слушать входящие вебхуки, запускаться по расписанию через Cron или активироваться вручную для быстрого тестирования. От триггера данные последовательно передаются в узлы действий, которые выполняют основную бизнес-логику. |
| **Сцена 3: Крупный план настройки узла n8n, демонстрация ввода выражения с использованием `{{ $json }}`.** | Воркфлоу требуют динамических данных. Вместо статического текста мы используем мощный синтаксис выражений n8n. Введя `{{ $json }}` и используя стандартную точечную нотацию JavaScript, вы можете динамически извлекать и подставлять переменные из любого предыдущего шага. |
| **Сцена 4: Анимация схемы воркфлоу, разделяющейся на несколько веток через узлы IF и Switch, а затем объединяющейся через Merge.** | Реальная автоматизация требует ветвления. Мы применяем узлы IF и Switch, чтобы направлять потоки данных по разным сценариям на основе строгих логических правил. А когда параллельные задачи выполнены, узлы Merge позволяют элегантно объединить эти ветки обратно в единый поток данных. |
| **Сцена 5: Сплит-экран: слева вебхук принимает данные формы, справа узел Gmail отправляет приветственное письмо.** | Перейдем к практике. Мы соберем автоматический приветственный пайплайн: перехватим отправку веб-формы через триггер, проверим наличие всех полей, залогируем недостающие данные и используем узел Gmail для отправки персонализированного письма нашему новому клиенту. |
| **Сцена 6: Экран вкладки Credentials в n8n с размытым API-ключом и значком щита/замка.** | Безопасность — это не опция. Вы никогда не должны прописывать API-ключи или Bearer-токены жестко в HTTP-запросах. Платформа n8n предоставляет встроенное, зашифрованное хранилище Credentials для безопасной настройки авторизации и работы с токенами OAuth2. |
| **Сцена 7: Блок кода на Python с выделенными ключевыми словами аннотаций типов и конструкцией `try-except-finally`.** | Когда визуальных узлов недостаточно, в дело вступает Python. При проектировании функций обязательно используйте аннотации типов для строгих контрактов данных с агентом. А чтобы защитить систему от падений, оборачивайте логику в блоки `try-except-finally` для безопасного перехвата ошибок. |
| **Сцена 8: Схема самовосстанавливающегося цикла, где красная нода ошибки направляется в LLM для анализа и автоисправления.** | В продакшене API неизбежно падают. Чтобы система выжила, мы используем глобальные обработчики ошибок и настройки Retry On Fail. Мы проектируем пути самовосстановления, где диагностическая петля перехватывает сбой и отправляет его ИИ-агенту с инструкцией по динамическому автоисправлению. |
| **Сцена 9: Диаграмма конечного автомата, показывающая узлы базы данных для сохранения состояния и узел Wait для паузы.** | Простые пайплайны отрабатывают за секунды, но когнитивные задачи агентов требуют архитектуры переходов состояний. Для долгоживущих процессов мы сохраняем контекст в базу данных и используем узлы Wait, чтобы заморозить воркфлоу, не расходуя ресурсы сервера, пока человек не подтвердит действие. |
| **Сцена 10: Ведущий в кадре, рядом графическое сравнение: прямая линия (DAG) против зацикленного графа (LangGraph).** | Наконец, важно понимать пределы no-code оркестраторов. n8n — это направленный ациклический граф, идеальный для линейных процессов. Но когда ваша архитектура требует глубоких, автономных когнитивных циклов `while-true`, вам придется выйти за рамки холста и перейти к фреймворкам вроде LangGraph. |

Сценарий готов к использованию в телесуфлере! Желаете ли вы перейти к разработке практических заданий или квиза для проверки знаний по третьей неделе?
