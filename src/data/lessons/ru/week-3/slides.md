# Презентация: Основы n8n и первый рабочий процесс

📊 Слайд 1. Блок 1 (ИИ-Инженер / Автоматизация): Инфраструктура n8n — развертывание self-hosted n8n в Docker и подключение PostgreSQL как БД.
* **Концепт дизайна:** Тёмная тема (Dark mode), схема архитектуры сервера. Иконки Lucide: `Server`, `Database`.
* **Ключевые факты:**
 * Self-hosted версия дает полный контроль над данными и снимает облачные ограничения на количество выполнений.
 * Развертывание производится через `docker-compose` с выделением томов (volumes) для сохранения данных.
 * PostgreSQL используется в качестве надежного бэкенда для хранения логов и учетных данных вместо дефолтной SQLite.

---
📊 Слайд 2. Блок 2 (ИИ-Инженер / Автоматизация): Холст и триггеры n8n — интерфейс платформы, типы триггеров (Manual, Webhook, Schedule) и экшены.
* **Концепт дизайна:** Светлая тема (Light mode), мокап визуального редактора n8n. Иконки Lucide: `Zap`, `Play`, `MousePointerClick`.
* **Ключевые факты:**
 * Рабочий процесс (Workflow) строится в визуальном Editor UI и всегда начинается с узла-триггера.
 * Основные триггеры: Webhook (по внешнему запросу), Schedule (по расписанию/cron) и Manual (ручной запуск для тестов).
 * Узлы действий (Action nodes) выполняют интеграции с сервисами и передают данные слева направо.

---
📊 Слайд 3. Блок 3 (ИИ-Инженер / Автоматизация): Передача данных и Expressions — синтаксис выражений n8n и работа со встроенными переменными.
* **Концепт дизайна:** Тёмная тема (Dark mode) с блоком кода. Иконки Lucide: `Braces`, `Code`.
* **Ключевые факты:**
 * n8n оперирует данными в формате JSON.
 * Expressions (выражения) позволяют динамически подставлять значения вместо жестко заданных (Fixed) данных.
 * Доступ к данным предыдущих нод осуществляется через JavaScript-синтаксис и встроенную переменную `{{ $json }}`.

---
📊 Слайд 4. Блок 4 (ИИ-Инженер / Автоматизация): Логическое ветвление — ноды IF, Switch и Merge (Wait, Pass-through, Combine).
* **Концепт дизайна:** Разделенный экран со схемой развилки (Flowchart). Иконки Lucide: `GitBranch`, `GitMerge`.
* **Ключевые факты:**
 * Узлы `IF` и `Switch` направляют поток данных по разным веткам на основе заданных логических условий.
 * Узел `Merge` позволяет объединить данные из нескольких параллельных веток обратно в единый поток.
 * Поддерживаются режимы ожидания (Wait), сквозной передачи (Pass-through) и комбинирования данных.

---
📊 Слайд 5. Блок 5 (ИИ-Инженер / Автоматизация): Практикум: Автоматический приветственный пайплайн — сборка процесса отправки email-оповещений.
* **Концепт дизайна:** 3-шаговая визуализация воркфлоу. Иконки Lucide: `Mail`, `Send`, `Bot`.
* **Ключевые факты:**
 * Пайплайн стартует с триггера (например, новой формы или входящего письма через Gmail Trigger).
 * Данные фильтруются, парсятся и могут быть обогащены с помощью ИИ.
 * Завершающий шаг — маршрутизация: отправка персонализированного приветственного Email и уведомления в Slack.

---
📊 Слайд 6. Блок 6 (ИИ-Инженер / Автоматизация): Авторизация внешних API в n8n — настройка Credentials, Bearer-токенов и интеграция OAuth2.
* **Концепт дизайна:** Тёмная тема (Dark mode), схема хранилища ключей. Иконки Lucide: `Key`, `ShieldCheck`.
* **Ключевые факты:**
 * Учётные данные (Credentials) безопасно хранятся в зашифрованном виде отдельно от холста воркфлоу.
 * Для кастомных интеграций через HTTP Request используются заголовки авторизации (например, `Bearer {token}`).
 * API-ключи никогда не должны прописываться жестко в теле запроса в целях безопасности.

---
📊 Слайд 7. Блок 7 (Python-разработка): Проектирование функций, аннотации типов и обработка системных ошибок через try-except-finally блоки.
* **Концепт дизайна:** Окно терминала/IDE. Иконки Lucide: `Terminal`, `AlertTriangle`.
* **Ключевые факты:**
 * Python-скрипты обеспечивают детерминированную логику и служат "руками" для AI-агентов.
 * Аннотации типов (Type hints) формируют строгие контракты (JSON-схемы) для корректного вызова инструментов языковой моделью.
 * Блоки `try-except` перехватывают сбои, а `finally` гарантирует очистку системных ресурсов независимо от результата.

---
📊 Слайд 8. Блок 8 (Создатель ИИ-Агентов / Агенты & Harness): Проектирование путей самовосстановления и обхода ошибок в визуальных воркфлоу n8n.
* **Концепт дизайна:** Циклическая схема. Иконки Lucide: `RefreshCw`, `Activity`.
* **Ключевые факты:**
 * Настройка «Retry On Fail» с экспоненциальной задержкой защищает пайплайн от временных падений API и лимитов.
 * Глобальный узел `Error Trigger` перехватывает сбои со всего инстанса n8n для отправки уведомлений администраторам.
 * Агенты используют диагностическую петлю: перехватывают ошибку и отправляют её в LLM для динамического автоисправления.

---
📊 Слайд 9. Блок 9 (Создатель ИИ-Агентов / Агенты & Harness): Архитектура переходов состояний (State Transitions) в визуальных графах автоматизации.
* **Концепт дизайна:** Диаграмма конечного автомата (FSM). Иконки Lucide: `Network`, `PauseCircle`.
* **Ключевые факты:**
 * Для долгоживущих задач (часы/дни) необходимо сохранять глобальное состояние (State) в базу данных между сессиями.
 * Узел `Wait` приостанавливает выполнение воркфлоу, не расходуя ресурсы процессора, для реализации Human-in-the-Loop.
 * Маршрутизатор состояний (Switch) детерминированно переводит процесс на следующий этап на основании сохраненного статуса.

---
📊 Слайд 10. Блок 10 (Создатель ИИ-Агентов / Агенты & Harness): Линейные процессы vs циклические графы: пределы no-code оркестраторов.
* **Концепт дизайна:** Сравнение прямой линии и зацикленного графа. Иконки Lucide: `ArrowRight`, `RefreshCcw`.
* **Ключевые факты:**
 * n8n отлично подходит для линейных процессов (DAG) с предсказуемым потоком данных, где контроль находится у инженера.
 * Циклические графы (Агенты) передают контроль LLM, позволяя ей свободно выполнять инструменты в цикле `while-true`.
 * Для сложных мульти-агентных архитектур и глубокой рекурсии необходимо переходить на code-first фреймворки (например, LangGraph).
